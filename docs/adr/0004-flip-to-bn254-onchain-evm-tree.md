# Flip curve to BN254 + maintain the commitment tree on-chain on EVM

We chose to maintain the authoritative commitment Merkle tree **on-chain on Sepolia** (fork `tornado-core` `MerkleTreeWithHistory.sol`), rather than off-chain in the relayer, to make EVM the source of truth for the commitment set and keep a path toward a trustless light-client root verification. The relayer anchors a recent EVM root into the Soroban pool's root window; withdrawal proofs prove membership against a root the Soroban side trusts.

That requires a SNARK-friendly hash **in Solidity** — and BN254 is the only field with a working off-the-shelf Solidity Poseidon (`poseidon-solidity`, zk-kit). A BLS12-381 `poseidon255` in Solidity does not exist anywhere; keeping BLS12-381 would mean hand-porting it (no repo, no test vectors, audit-grade, worse gas with no EIP-2537 precompile). So we **flip the curve to BN254** (supersedes ADR-0003): one Poseidon instantiation now spans circom + Solidity + Rust + Soroban, eliminating the cross-surface mismatch trap. BN254 Groth16 also verifies on Soroban today (CAP-0074 g1 ops + CAP-0080 MSM/Fr, P25/P26), and `NethermindEth/stellar-private-payments` ships a BN254 Groth16 verifier already deployed on testnet plus matching circom circuits and ceremony tooling — retiring most verifier risk.

**Keystone / consequence:** the hash must be byte-identical across all three surfaces. NethermindEth uses **Poseidon2** (HorizenLabs `zkhash` `POSEIDON2_BN256_PARAMS_2`, compression `P(l,r)[0]+l`), **not** classic Poseidon1 — and stock `poseidon-solidity`/circomlib are Poseidon1, which will not match. Day-1 spike must assert a shared `hash([1,2])` test vector across circom, Rust, and Solidity before any other work. Fallback ladder if Poseidon2-in-Solidity proves fiddly: Poseidon1 + `poseidon-solidity` for the EVM tree with a dedicated membership sub-circuit → stock tornado MiMCSponge. Pin `soroban-sdk = 26` (`Bn254Fr` rename + CAP-0080 bindings). The on-chain EVM tree's verified cost (~420k gas / depth-20 deposit) is accepted; the Soroban verify cost has no published number — measure it on testnet day-1.
